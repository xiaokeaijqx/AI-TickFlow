# TickFlow

> **你的 AI 智能体，一块悬浮的指挥面板。**
> 用 Markdown 写任务。看 AI 一条条干完。复选框自己打勾。

[English](./README.md) · 中文

TickFlow 是一个常驻置顶的 macOS 悬浮挂件，把一份普通的 `task.md` 变成你 AI 编码智能体的实时任务中枢。把任务丢进去，点执行，TickFlow 就会在真实的终端会话里驱动 **Claude、Codex 或任意 CLI 智能体** 逐条完成——一边实时勾掉复选框，一边让你腾出手去做别的事。

不用复制粘贴。不用盯着看。不用来回切窗口。只有一块悬在工作区上方的小玻璃面板，安静地把活儿干完。

---

## ✨ 为什么用 TickFlow

**它是真的替你跑智能体。**
TickFlow 会拉起一个真实的 `tmux` 会话、启动你的智能体、把 prompt 粘进去，再把终端输出实时回流到内置日志面板。你只管看进度，不用自己开车。

**一窗一项目，多个一起跑。**
一个窗口里让 Claude 推进功能 A，另一个窗口里让 Codex 同时啃重构——每个窗口绑定自己的 `task.md`、自己的智能体、自己的权限。按 **⌘N** 开个新项目，让它们赛跑。

**Markdown 才是唯一事实来源。**
任务存在你自己的文件里。TickFlow 盯着它（500ms 轮询，编辑器友好），改动双向同步——在界面里勾一个框，或在编辑器里改一行，另一边立刻更新。

**排好队，走人。**
把任务分批，再把批次排队，TickFlow 会自动逐批推进——**每个队列**一完成就用声音提醒你。回来看到的是一摞干完的活，而不是卡住的提示符。

**它不会悄无声息地挂掉。**
每个智能体都有独立的停滞看门狗，发现智能体安静太久会提醒你——而且绝不会在你背后偷偷杀掉会话。

## 🚀 工作原理

1. **选一个 `task.md`** —— 或者开多个，一窗一个
2. TickFlow 把每一行 `- [ ]` 解析成实时清单
3. **选任务 → 执行** —— 自动分批并派发给你的智能体
4. TickFlow 在 `tmux` 会话里拉起智能体并 **自动粘贴 prompt**
5. 智能体干活，按 `DONE N` 逐任务上报，TickFlow 帮你勾框
6. 🔔 **每个队列完成时** 触发声音 + 通知 —— 排好队就可以不管了

智能体通过轻量标记上报进度——不靠脆弱的屏幕抓取：

| 标记 | 含义 |
|---|---|
| `DONE N` | 当前批次的第 N 个任务完成 → TickFlow 勾掉它 |
| `WAIT_APPROVAL` | 智能体需要你确认 → 你会收到通知，可批准/拒绝 |
| `ALL_TASKS_COMPLETED` | 全部完成 |

每个批次有独立的哨兵锚点来定位标记，即使终端滚动缓冲区滑动也能保持正确——经受过偏移漂移的考验。

## 🎛 功能特性

- **🪟 多窗口项目** —— 一窗一个 `task.md`，各有独立智能体与配置；用 **⌘N** 或设置页开新窗
- **🤖 自带智能体** —— **Claude**、**Codex** 或任意 **自定义命令**，可 *按项目* 配置
- **🔓 按项目权限** —— 可逐窗开启 Claude 的 `--dangerously-skip-permissions`，无人值守批量跑
- **🛰 实时终端面板** —— 智能体完整日志流进挂件，可选按键透传
- **📋 批次队列** —— 选择 → 分批 → 排队；自动逐批推进，**每队列完成都提醒**
- **🔁 Markdown 双向同步** —— 界面 ↔ `task.md`，双向，实时
- **👀 编辑器友好的文件监听** —— 500ms mtime 轮询，比 `fs.watch` 更可靠
- **🧭 7 种任务状态** —— TODO · RUNNING · DONE · FAILED · PAUSED · STOPPED · QUEUED
- **🐶 停滞看门狗** —— 每个智能体独立的空闲检测，只提醒不杀会话
- **🪄 折叠模式** —— 缩成 136×86 的小胶囊显示实时进度，完成时有动画
- **↩️ 撤销删除** —— 任意删除的任务有 6 秒恢复提示
- **🔔 自定义提示音** —— 完成与审批可选任意 macOS 系统音
- **⌨️ 全局快捷键** —— 一键召唤/隐藏所有窗口（默认 **⌘⇧T**）
- **♻️ 一键重启** —— 在设置页重启智能体（切换模型/配置后）或整个应用

## 🏗 架构

```
┌──────────────────────────────────────────────────────────────┐
│                    Electron 主进程                              │
│                                                                │
│   ┌──────────────┐   ┌──────────────┐   ┌──────────────────┐  │
│   │ 窗口注册表     │   │ 按文件监听器   │   │ 按会话看门狗      │  │
│   │ (一窗一项目)   │   │ (500ms 轮询)  │   │ (停滞检测)        │  │
│   └──────┬───────┘   └──────┬───────┘   └────────┬─────────┘  │
│          │                  │                     │            │
│   ┌──────┴──────────────────┴─────────────────────┴────────┐  │
│   │            tmux 智能体会话（按项目隔离）                  │  │
│   │      claude / codex / custom  ←  粘贴 prompt  →  日志    │  │
│   └─────────────────────────────────────────────────────────┘  │
│                            │                                    │
│                  task.md 文件（磁盘，唯一事实来源）             │
└─────────────────────────────┬──────────────────────────────────┘
                              │ IPC（contextBridge，按窗口）
┌─────────────────────────────┴──────────────────────────────────┐
│                       渲染层（React 18）                        │
│   App.tsx · TaskList · AgentPanel · SettingsPanel · taskStore   │
│        每个窗口 = 独立进程、独立 store、独立绑定文件             │
└──────────────────────────────────────────────────────────────┘
```

每个窗口都是绑定单一文件的独立渲染进程，通过同步启动握手建立——所以两个项目永远不会互相干扰彼此的状态、智能体或监听器。

## 🧰 技术栈

- **Electron 31** —— 无边框、透明、置顶的玻璃拟态窗口
- **React 18** —— 基于 `useSyncExternalStore` 的可变闭包 store（无额外状态库）
- **TypeScript 5** —— 全程 strict 模式
- **TailwindCSS 3.4** —— 工具优先 + 自定义 `tick` 设计令牌
- **tmux** —— 真实终端会话，真正地执行智能体
- **Vite 5** —— 渲染层打包；主进程用 `tsc`

## 🛠 开发

```bash
npm install
npm run electron:dev     # Vite 开发服务 + Electron
npm run typecheck        # TypeScript 检查（渲染层）
npm run electron:build   # 生产 .app + .dmg
```

**环境要求：** macOS · Node.js ≥ 18 · `tmux` · 智能体 CLI（`claude` / `codex` / …）已在 `PATH`
